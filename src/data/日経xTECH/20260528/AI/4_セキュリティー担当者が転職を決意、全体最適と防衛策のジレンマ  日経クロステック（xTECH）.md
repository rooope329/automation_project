# セキュリティー担当者が転職を決意、全体最適と防衛策のジレンマ | 日経クロステック（xTECH）

　コンサルティングファームでサイバーセキュリティー領域の人材採用が活発になっている。セキュリティーがもはや情報システム部門だけの課題ではなく、経営に直結する論点として扱われるようになってきたためだ。コンサルティング業界におけるセキュリティー職種への転職について、インディードリクルートパートナーズの佃翔太氏が解説する。

　昨今、サイバー攻撃によって事業やサービスが止まる事案が相次ぎ、多くの企業で危機感が高まっています。これまでセキュリティー対策の中心は、システムインテグレーターや専門ベンダーが担ってきました。ただ最近は、コンサルティングファームに相談する企業も増えています。経営課題としてセキュリティー上の論点を整理し、経営層へのアドバイスから製品導入などの実行支援まで一気通貫で関わる点でコンサルティングファームが評価されているためです。海外の先進事例を日本企業向けに翻訳し、顧客の意思決定につなげるといった役割も期待されています。

　コンサルティングファームはセキュリティー分野のニーズに対応するため、体制の整備を急いでいます。コンサルティングやセキュリティー分野の経験者に限らず、企業のガバナンス担当など隣接領域からの人材獲得にも踏み込んでいるのです。もともとこの領域は専門人材の母数が限られており、経験者だけで組織を強化するのは難しい状況です。そのため企業側も「どんな経験を持つ人ならセキュリティー担当者として育成できるか」という視点で採用を進めています。

　実際、コンサルティングファーム各社はシステムインテグレーターや事業会社で専門性を磨いてきた人材を、自社で育てる前提でセキュリティー担当者として迎え入れています。選考の目安の一つになるのが、CISSP（Certified Information Systems Security Professional）や情報処理安全確保支援士といったセキュリティー関連資格です。こうした土台があれば、コンサルティング業界で求められるクライアントワーク中心の経験がなくても、採用の可能性があります。

#### 「なかなか対策が進まない」もどかしさから転職へ

　一方、セキュリティーコンサルタントに転職する個人の側にはどんな動機があるのでしょう。システムインテグレーター出身者からよく聞くのは、提案内容に対する制限のもどかしさです。顧客の課題に最適な選択肢が見えていても、自社やパートナー企業の製品を優先せざるを得ないことがあります。より納得感のある提案をしたいと考え、製品を持たないコンサルティングファームに魅力を感じる人は少なくありません。

　事業会社の担当者からは「セキュリティー投資が後回しになりやすい」という悩みを聞きます。セキュリティー対策は本来、事業継続や企業価値にも関わる重要なことです。ところが企業の全体最適を目指すと、投資対象としては優先されないといった構造があるようです。事業会社で感じるこうした歯がゆさが、コンサルティング業界で積極的な変革の現場に関わりたいという動機につながっています。

　象徴的なのが、大手インフラ企業でセキュリティー部門の管理職を務めていたAさん（50代）の事例です。複数の施策を提案しても、経営会議で通るのは一部だけ。セキュリティーの重要性を訴えても意思決定のスピードはなかなか上がらず、自社の方針を変えていくことに限界を感じていました。

　どうするか悩んだAさんは他社のセキュリティー責任者などと情報交換し、社外でネットワークを築いていきました。他の責任者との話の中で自社と同じような課題を抱える企業が多いと理解したAさんは、やがて外から企業の変革を支援する側に回りたいと考え、転職活動に至りました。

## 2ページ目

　Aさんが選んだのは、セキュリティー担当の組織強化に力を入れている日系コンサルティングファームX社でした。筆者はヒアリングを通じ、X社が「インフラ業界へのビジネス拡大を目指しており、同業界の知見や人脈を持つ人材を求めていること」を知ってAさんに提案したのです。Aさんにとっては専門知識を生かせるだけでなく、新しい案件の開拓に携われる可能性が高い点が魅力だったといいます。

　選考ではAさんが事業会社で手がけた長期にわたるプロジェクトの推進や、社内の様々な部署との調整、社外パートナーとの協業といった実績も評価されました。プロジェクトマネジャーとして短期間では成果が見えない案件を担当し、PDCAを回し続けた経験がコンサルティング業界でも生かせると考えられたのです。実際、転職後は専門性を生かしながら中核ポジションで活躍し、インフラ業界からの新たなコンサルティング案件も獲得できたそうです。

　実はAさん本人は転職活動を始めた当初、選考過程でプロジェクトマネジャー経験が強みになるとは思っていなかったといいます。しかし、大手の事業会社で稟議から決裁までの流れを理解し、誰を巻き込めば意思決定が進むかを見抜くといった経験は、他の業界でプロジェクトを推進する際にも役立ちます。発注側として外部パートナーと協業した経験は、コンサルタントとして受注側に回っても生かせるでしょう。

　こうした本人も自覚していない価値ある経験を、採用側の企業が理解しやすい言葉に「翻訳」できるかどうかで、新たなキャリアが広がる可能性が高まります。企業側が評価するポイントと、求職者の経験双方を見比べつつ整理することが大切です。

#### 品質管理とセキュリティーの意外な共通点

　X社のようにセキュリティーコンサルタントの育成を急ぐコンサルティングファームでは、セキュリティー分野の経験者以外にも門戸を開きつつあります。実際、あるコンサルティングファームのセキュリティービジネス統括担当は「システム開発などで一定レベルの経験を持つ人であれば採用候補になる。セキュリティーの知見は入社後にインプットできる環境を用意している」と語ります。

　中でも筆者が注目しているのが、品質管理やソフトウエアテストの経験者です。最近は、ソフトウエアの品質評価・品質管理に携わってきたエンジニアの方々から、「AI（人工知能）の台頭により今後のキャリアの方向性に不安がある」といった相談を受けることがあります。品質管理とセキュリティーは一見離れて見えますが、共通点もあります。象徴的なのが「Wモデル」の考え方です。Wモデルとはテストの実施を待たず、初期の段階からテスト計画の作成に取り掛かる手法を指します。Wモデルに基づき、品質管理では開発と並行してテスト設計を進め、品質を高めていきます。問題が起きてから対応するのではなく、未然に防ぐという発想は、セキュリティーのアプローチと重なります。

　もちろん、誰でもすぐに転身できるわけではありません。しかし、例えば開発経験が2～3年程度以上あり、品質管理にも携わってきた中堅層であれば、現在の転職市場の状況ならセキュリティー分野のキャリアが開ける可能性があります。実際に、品質管理の経験を積んだ30代のエンジニアが、コンサルティングファームでセキュリティー担当として選考に進んだ事例もあります。

　なお、現実には「セキュリティー人材」という言葉が指す役割は多様です。セキュリティーに関する企画、ガバナンス、設計、開発、運用、監視、インシデント対応など、様々な業務が含まれています。例えば米国立標準技術研究所（NIST）が公開している「Workforce Framework for Cybersecurity（NICE Framework）」では、求められるセキュリティー人材像が約40種類に細分化されています。幅広い力が求められる仕事と言えるでしょう。

　一見、大変そうな業務ではありますが、視点を変えると先ほど紹介したAさんの事例のように、幅広い業務の中で思わぬ経験が評価される可能性もあります。Aさんのように「これから自分がしたい経験」を検討し、「これまで習得したスキル」とマッチしそうならセキュリティー分野でのキャリアを検討してみてはいかがでしょう。
