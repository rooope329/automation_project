# SBI新生銀は「AWSファースト」、約3年で障害半減　鍵は共通基盤の整備 | 日経クロステック（xTECH）

　システムの過半数は米Amazon Web Services（アマゾン・ウェブ・サービス、AWS）のクラウドへ。軽微なものを含む社内外のシステム障害は3年で半減――。SBI新生銀行は全社でのクラウドシフトを推進している。その方針の下支えとなったのは、クラウド利用を前提に整備したAPI（アプリケーション・プログラミング・インターフェース）などの共通基盤だ。

　「SBIハイパー預金」など独自のリテール戦略を展開するSBI新生銀行。これを支えるために同行は、「システムアーキテクチャー」と「ITガバナンス」の整備を両輪で進めてきた。

　このうちシステムアーキテクチャーの整備では、SBIホールディングス（SBIHD）の傘下に入る以前からAWSへのクラウドシフトを進めている。2026年5月時点で、約200ある全システムのうち約110をクラウドに移行した。併せて2022年11月には米New Relic（ニューレリック）の可観測性ツール「New Relic」も導入。軽微なものを含む社内外の年間システム障害件数は、2022年から2025年の3年間で半減したという。

　両輪のもう一方、ITガバナンスの整備については、2020年に発生したシステム障害を契機に経営とITの連携を強化した。

#### 会社全体で「クラウドファースト」を掲げる

　SBI新生銀行がAWSへのクラウドシフトを開始したのは、SBIグループ入り以前の2020年2月だ。子会社を含む新生銀行グループ全体でクラウドを使うことで、固定費の見直しやコスト削減、将来のサービス拡充に備えた機動性や拡張性の確保を図る狙いだった。

　SBI新生銀行がクラウドシフトを推進できた要因は3つある。1つめがAWSへのクラウドファーストの方針を会社全体で掲げた点だ。オンプレミスのシステムを更改する場合、コストや難易度などで移行が困難な場合を除き、AWSへの移行を基本とする運用を固めた。

　2つめが現行の勘定系システム「FLEXCUBE」の存在だ。特集の第1回で詳述した、米Oracle（オラクル）が子会社を通じて提供する金融機関向けパッケージだ。ユーザー企業にとって内部構造を把握しにくいため、勘定系システムそのものの扱いには苦労していた。

　だがFLEXCUBEはメインフレームではなくオープンシステムで稼働するため、クラウドシフトという文脈では「結果的に下地になった」（出島良次グループIT統括部長）。メインフレーム中心の構成に比べ、外部連係や改修の自由度が高い。人材面でもメインフレーム専任のエンジニアがいなかったため、クラウドシフトに際し、開発や運用担当のエンジニアをオープン系やクラウド関連業務に集約しやすかった。

　3つめは各システムが利用する「共通基盤」を個別システムの開発や活用に先立って事前に整備したことだ。セキュリティーや監視、外部接続など、各システムに共通で備えておくべき機能をクラウド上に整備した。SBI新生銀行の白方貴史システム運用部長兼グループIT統括部統轄次長は「各部門が個別にクラウドを使い始めると、システムが乱立し、セキュリティー水準もばらつく。そこで、まずはルールと仕組みを共通化する基盤を整えた」と語る。
