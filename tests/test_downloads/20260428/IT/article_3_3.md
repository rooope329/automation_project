# 26年共通テストの情報Ⅰ大幅難化、話題の「くまちゃん問題」に感じた惜しさ | 日経クロステック（xTECH）

2026年度の大学入学共通テスト「情報Ⅰ」は大幅に難化した――。大学入試センターが発表した平均点は56.59点で、昨年の69.26点から大きく下がった。試験直後から、SNSやブログなどで難化を指摘する受験者の声が目立った。大手予備校の評価もおおむね同じだ。河合塾は「難化」とし、マーク数が昨年より9増えたことや、幅広い知識を基に考察する問題が出たことを指摘。駿台予備学校・ベネッセコーポレーションも「昨年より難化」とし、深い思考を要する問題があったと分析した。

　筆者も今年の情報Ⅰを解いてみた。今回は情報システムやプログラミング寄りの大問1から大問3までを解き、昨年満点を取れなかったデータ活用の大問4は対象から外した。自己採点では全問正解だった。

#### 「くまちゃん問題」に頭が混乱

　解いた中で最も頭を使わされたのが、ネット上で「くまちゃん問題」などと話題になった第2問Bである。

　この問題では、ゲーム画面の背景画像にキャラクター画像を重ねる処理を考えさせる。画像の各点は4ビットのグレースケールで表され、白は1111、黒は0000となる。受験生はOR演算やAND演算、NOT演算の真理値表を見ながら、キャラクター画像の白い背景を透過させ、背景画像に重ねるにはどの処理をすればよいかを追っていく。

　理屈は分かる。白と黒をビット列として扱い、OR演算やAND演算の性質を使えばよい。だが、実際に解いてみると、頭の中で画像を抜き出し、重ね、さらに演算結果を追う必要がある。特に「セ」「ソ」「チ」に当たる部分は、前段の理解を誤ると後続の空欄も連鎖的に間違えやすい。1つの判断ミスが複数の失点につながる。筆者も途中で何をANDして、何をORしているのか分からなくなりそうだった。大量失点してもおかしくなかった。

## 3ページ目

#### 30年前のスーパーインポーズの思い出

　実はこの問題を見て、少し懐かしくなった。今は大学のサークルの新入生歓迎の時期だが、筆者も大学時代に所属していたスキーサークルで、プロモーションビデオの作成を手伝ったことがある。某大手企業の映像処理装置を借りる機会があり、そこで初めて本格的な画像処理を体験した。

　映像の上に文字や別の映像を簡単に重ねられることに驚いた。操作方法に詳しい同級生にこの機能について尋ねると、「スーパーインポーズ」だと教えてくれた。当時の筆者には、映像の重ね合わせが急に身近になった瞬間だった。

　もちろん、当時の筆者はAND演算やOR演算を意識していたわけではない。装置を操作すれば、画面上でそれらしいことができたにすぎない。今回のくまちゃん問題も、問題文に「スーパーインポーズ」という言葉は出てこない。ただ、画像の一部を透過させ、別の画像に重ね合わせる仕組みに通じる問題だったとは言えるだろう。筆者は、出題者が受験生にその仕組みを知ってほしいと考えたのではないか、と勝手に想像した。

　昨年の情報Ⅰにも、教科書にそのまま載っていない題材が出た。7つのLEDで数字などを表現する「7セグメントLED」の問題や、マウスカーソルの移動時間を考える問題である。後者は、後で調べると「フィッツの法則」と呼ばれる考え方に近かった。今年のくまちゃん問題も、その流れにある。教科書の用語を覚えたかではなく、実際のITに近い題材を使って、情報の考え方を問おうとしている。

#### 実社会との接続をどう見せるか

　一方で、少し引っかかった点もある。問題文では、画像の透過や重ね合わせに論理演算を使うこと自体は説明されていた。だが、情報ⅠでAND演算やOR演算を学ぶ場面は、多くの場合、論理回路や真理値表の説明に近いのではないか。そこからいきなり画像処理に飛ぶと、受験生には「論理演算の応用」を考える問題ではなく、「初見の処理を時間内に読み解く問題」に見えた可能性がある。

　筆者は、この題材自体は面白いと思う。画像の一部を透過させ、別の画像に重ねる処理に、AND演算やOR演算が関係していると知ることには意味がある。ただ、共通テストの限られた時間で、しかも前段の理解を誤ると後続の空欄まで連鎖的に崩れる構造にする必要があったのか。1つの判断ミスが大量失点につながりかねない点は、試験問題として少し気になった。

　情報Ⅰには、相変わらず気になる点もある。今年のプログラミング問題でも、共通テスト用のプログラム表記やローマ字変数名には、実際のプログラミングとは少し違う読みづらさが残っていた。第3問では「taiken（体験時間）」「Touchaku（到着時刻）」「Kaishi（開始時刻）」「Shuryou（終了時刻）」「kyakusu（来訪者数）」といった変数名や配列名が出てくる。昨年も「buinsu（部員数）」「Akibi（空き日）」といった変数名が独特で、コードを読みにくくしていた。ここは過去の違和感が残った部分だ。

　2026年度の情報Ⅰは、確かに難化した。ただ、本質は難しくなったことだけではない。画像処理や情報システム、プログラミングなど、IT前提社会で必要になる仕組みを、受験生に考えさせようとしている。その方向性は悪くない。筆者が「惜しい」と感じたのは、くまちゃん問題のような良い題材が「初見の処理を時間内に読み解く問題」に見えてしまう点だ。普段の授業や教材で、論理演算が画像処理につながると学べていれば、くまちゃん問題の見え方も変わったはずだ。情報Ⅰは「入試の新科目」ではなく、「社会に出て役立つ科目」としてもっと伝わりやすくなるのではないか。
