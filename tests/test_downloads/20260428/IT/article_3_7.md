# 日本IBMがAI駆動開発向けコンテキストを顧客に提供、35％の工数削減目指す | 日経クロステック（xTECH）

2027年以降にはシステム開発プロジェクト全体において、35％の工数削減や30％の開発期間短縮を目指す――。

　日本IBMはAI（人工知能）を使い、エンタープライズ向けのシステム開発や運用、プロジェクト管理などを支援する「IT変革のためのAI」を重要な戦略の柱に据えている。この取り組みの一環として、生成AIを活用したシステム開発向けの標準コンテキスト「AI Lifecycle Shared Engineering Artifacts（以下ALSEA、アリーシア）」の提供を先行プロジェクト向けに開始した。

　ALSEAはシステム開発を支援する生成AIに対してコンテキストを提供するマークダウン形式のファイル群である。AIがシステム開発を自律的に進められるように、日本IBMが蓄積してきた形式知やエンジニアの暗黙知などをまとめてコンテキストとして言語化・構造化した。日本IBMが提供するAI駆動の開発支援ツール「IBM Bob」と組み合わせて使うことを想定する。

AI駆動の開発支援ツール「IBM Bob」

（画像：日本IBM）

　IBM Bobは自然言語の入力内容をAIが理解し、ユーザーの意図に沿ったコードなどの成果物を出力する。同様のツールは他社も提供を始めているが、IBM Bobの特長は安全性や成果物の再現性などを重視している点が挙げられる。ALSEAとIBM Bobの組み合わせにより、冒頭のような効果を見込む。

#### エンタープライズ向けに特化

　日本IBMはAI駆動開発の区分を3つに分けて整理している。主に非エンジニアがAIに自然言語で指示をしてコードを生成させる「バイブコーディング」、高度なスキルを保有するエンジニアがAIの計画を確認・修正しながらAIにコードを生成させる「ハイブリッド」、重要なシステム開発を保守するエンジニアが仕様書を作成して技術負債が少ないコードを生成させる「仕様駆動開発」である。バイブコーディングはプロトタイプ開発が中心で、仕様駆動開発はエンタープライズ向けのシステム開発を対象としている。

AI駆動開発の区分

（画像：日本IBM）

　日本IBMはエンタープライズ向けの仕様駆動開発をターゲットに据え、IBM BobやALSEAを開発している。日本IBMの高橋聡執行役員コンサルティング事業本部インダストリーサービス＆デリバリー統括は「バイブコーディングは技術者以外の人たちによる開発を可能にしたが、エンタープライズ向けのシステム開発で利用するには難しい」と指摘する。
